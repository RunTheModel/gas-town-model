# Modeling Gas Town

This repo contains artifacts created by modeling the Gas Town Convoy flow in Kinner, a new formal modeling language under development at [runthemodel.dev](https://runthemodel.dev).  They are being used to drive [a web simulator](https://gastown-demo.salmonfield-c02b39a2.eastus.azurecontainerapps.io/) you can play with today.

## What is Kinner?
Kinner is a modeling language that creates verifiable models and checks them with the TLA+ TLC model checker via the Kinner compiler known as Earhart. Kinner has not been released yet but will be released for preview very soon (likely before May 1 2026).

## What is Gas Town?
[Gas Town](https://github.com/gastownhall) is a new orchestration layer for AI coding agents. 

## What are these artifacts?
These artifacts were generated by pointing it at the gas-town repo and distilling out these into a model with 14 actors and four invariants, verified with TLS across almost ten million states.

These artifacts can be grouped into a few categories
* **Kinner Models (*.kinner.json*)**  - JSON files modeling actors discovered from the [Gas Town](https://github.com/gastownhall) code repository.
* **Python Simulator (bin/\*.py)** - generated Python files allowing you to step through the process of landing a convoy. This interactive demo is perhaps the clearest way available to see how work flows through Gas Town.
* **Web App (app.py)** - a Streamlit app providing a browser interface to the Python simulator. Currently driving [the web simulator](https://gastown-demo.salmonfield-c02b39a2.eastus.azurecontainerapps.io/).
* **Diagram (convoy-lands.dot)** - graphviz visualization rendered from the Kinner model. Paste into [edotor](https://edotor.net/) or use your favorite graphviz editor.

## What can I do with this stuff?

### Look at Kinner
Kinner is "just JSON" but it is a composable language that compiles down to TLA+ for verification. take a peek if you want an early preview of what it looks like.

### Look at a Diagram
Raw JSON is no fun. Kinner can generate graphviz diagrams based on generated models. This repo includes `convoy-lands.dot` - a graphviz diagram generated from the Kinner model. Open it in a DOT visualizer to see a depiction of a a mayor landing a Convoy, with the Mayor, Hooks, Convoys, and Polecats all working together.

### Drive the simulator online
Diagrams are better than JSON but they are static. Kinner generates interactive simulators based on the models you create with it. You can interact with a [Streamlit](https://streamlit.io/) demo of the convoy process at [the simulator website](https://gastown-demo.salmonfield-c02b39a2.eastus.azurecontainerapps.io/).  You can direct the various elements to take actions and follow the work through the system.

### Drive the simulator on your own machine
If you want to download and run the simulator, clone this repo 

#### Console Mode
Run `python convoy.py` for an interactive console experience (pick transitions by number, watch the state evolve, see invariants checked at every step). No dependencies beyond Python.

#### In a local browser
  For the browser UI, `pip install streamlit` and run
  `streamlit run app.py`.

### Rebuild the model
  To regenerate the simulator from the Kinner source, run
  `build.cmd` (requires the EARHART compiler).  If you want access to bits _right now_ drop us a line at [hello@runthemodel.dev](mailto:hello@runthemodel.dev)

## Why Gas Town?
  
Gas Town and Kinner are well-matched on several levels.

* **Kinner development** - As Kinner has matured in capability we are taking on bigger projects. Modeling Gas Town has already shaken out a number of bugs in Kinner, and the process of doing has inspired numerous other features. The structured AI-driven development workflow is similar in spirit to the workflow used to develop Kinner.

* **Keep Gas Town Honest** - Gas Town is a very quickly-evolving framework. The Kinner model here is based on a recent snapshot of the Gas Town source code.  We found several places where the docs are a bit stale. As things change this model will be updated.
  
* **Learn about Gas Town** - Gas Town is a relatively new framework with complex workflows using a stylized vernacular to describe it, and docs are a little thin. The authors found building this model and playing with the simulator to be the clearest way to understand how work flows through Gas Town.
